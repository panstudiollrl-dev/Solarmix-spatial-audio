# Solarmix Spatial Audio v0.1.0-alpha

This alpha marks Solarmix's first public open-source milestone as a Unity-based spatial audio validation instrument.

Solarmix is not positioned as a finished music app yet. This release establishes the project structure, active MeshRIR branch, documentation, and validation goals for testing spatial audio behavior with controlled orbital motion, timbre, envelope, distance, and open spatial audio datasets.

## Focus

- Active branch: `meshRIR`
- Platform focus: Unity Editor and iOS/headphone listening
- Primary use case: spatial-audio effect validation
- Current backend direction: MeshRIR-inspired interpolation with procedural physical-model source probes

## What's Included

- Unity project structure for the Solarmix orbital sound scene.
- Steam Audio baseline history preserved on `main` / `steam-audio`.
- Active MeshRIR branch with current mobile and interpolation work.
- Documentation for:
  - spatial-audio validation method;
  - dataset provenance;
  - AI-assisted development workflow;
  - contribution and issue reporting;
  - third-party notices and licensing boundaries.
- Public roadmap issues for validation scenes, MeshRIR fixtures, planet voice probes, iOS listening checks, and demo media.

## Validation Direction

Solarmix uses visible orbital motion as a repeatable listening scene. The project is designed to compare how spatialization algorithms respond to:

- circular, elliptical, flyby, figure-eight, static, and center-crossing motion;
- tonal, noisy, transient, resonant, broadband, and low-frequency source models;
- distinct envelope shapes;
- distance and room-energy changes;
- HRTF, BRIR, RIR, SRIR, HiFi-HARP, and MeshRIR-style datasets.

## Dataset Notes

No large downloaded spatial audio datasets are bundled in this release.

Dataset integration is documented in `docs/DATASETS.md`. Current and planned sources include:

- MeshRIR: https://github.com/sh01k/MeshRIR and https://zenodo.org/records/10852693
- HiFi-HARP: https://huggingface.co/datasets/whojavumusic/hifi_harp
- SOFA-format HRTF/BRIR/SRIR candidates, including CIPIC, SADIE II, and SONICOM

Each dataset adapter should document source URL, citation, license, redistribution terms, coordinate convention, sample rate, channel format, and fallback behavior.

## Known Limitations

- The active MeshRIR branch can run without downloaded datasets, but real MeshRIR fixture setup is still in progress.
- Demo screenshots/video are planned but not included in this alpha.
- The sound-design system is still being tuned through listening tests.
- Spatial continuity, distance scaling, and iOS clipping/limiting remain active validation topics.
- Build artifacts and app binaries are not distributed in this source release.

## Why This Release Exists

This alpha is a public checkpoint for the open-source workflow. It makes the project's purpose inspectable:

- artist-led spatial audio research;
- AI-assisted Unity/C# implementation;
- open dataset provenance;
- repeatable listening validation;
- branch-based comparison of spatialization approaches.
