# AI Collaboration Notes

Solarmix is directed by PAN STUDIO LLRL and developed through an iterative artist-agent workflow.

The core concept, spatial-audio goals, planet/orbit metaphor, sound-design preferences, interaction requirements, and listening feedback come from the project maintainer. AI coding agents have been used to implement Unity systems, refactor audio code, create branches, build iOS targets, inspect logs, draft documentation, and explore spatialization approaches.

## Why This Matters

Solarmix is a useful open-source case because it is not only a codebase. It is a record of how agentic coding can support a specialized creative-technical workflow:

- turn listening notes into concrete DSP and UI tasks;
- compare spatial audio algorithms across branches;
- rapidly prototype mobile builds for real headphone testing;
- document third-party dataset assumptions;
- convert artistic critique into reproducible validation scenes.

## Maintainer Role

The maintainer provides:

- artistic direction;
- sound-design goals;
- listening evaluation;
- dataset and research priorities;
- product and interaction judgment;
- final review of what sounds correct.

## Agent Role

Agents assist with:

- Unity/C# implementation;
- branch management;
- build automation;
- documentation;
- code review;
- bug investigation;
- issue and roadmap drafting.

## Contribution Standard

Agent-generated changes should be reviewed like any other contribution. A change is not finished until it is buildable, documented when needed, and evaluated through listening or a relevant validation scenario.
